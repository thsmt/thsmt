# hsmts2

Cloud Engineering / Infrastructure / AWS

AWS を中心に、ネットワーク・サーバー・Webアプリケーション基盤を Infrastructure as Code として整理しています。学習した構成をそのまま写すのではなく、CloudFormation テンプレート、構成図、パラメータ例、セキュリティ注意点、運用メモとして再現できる形にまとめています。

- 2025 Japan All AWS Certifications Engineer
- 2026 Japan All AWS Certifications Engineer
- 現在は AWS、IaC、Modern DevOps practices を中心に学習・整理中です。

## Representative Repositories

| Repository | Overview | Focus |
| --- | --- | --- |
| [aws-nwsv-lab](https://github.com/hsmts2/aws-nwsv-lab) | VPC + Web / DB 2層構成の基礎学習リポジトリ | VPC、Subnet、NAT Gateway、Security Group、CloudFormation |
| [aws-network-fundamentals-lab](https://github.com/hsmts2/aws-network-fundamentals-lab) | ALB + Webサーバー2台 + DBサーバー構成の再現 | ALB、EC2、MariaDB、WordPress、ネットワーク分離 |
| [aws-webapp-infra-lab](https://github.com/hsmts2/aws-webapp-infra-lab) | Railsアプリケーション基盤を想定したAWSインフラ構成 | RDS、S3、ElastiCache、Secrets Manager、Parameter Store、設計差分整理 |

## Strengths

- AWS ネットワーク、EC2、RDS、ALB、S3、ElastiCache などを組み合わせたインフラ設計
- CloudFormation による構成のコード化と再現性のある手順化
- セキュリティグループ、秘密情報管理、パラメータ管理を意識した構成整理
- README、構成図、デプロイ手順、補足メモを含めたドキュメント整備
- 手動構築した内容を IaC として置き換える学習・改善プロセス
